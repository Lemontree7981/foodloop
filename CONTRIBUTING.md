1. **Update documentation** if you're changing functionality

### PR Title Format

Use conventional commit format:
- `feat: add new feature`
- `fix: resolve bug with X`
- `docs: update README`
- `style: format code`
- `refactor: restructure X component`
- `test: add tests for X`
